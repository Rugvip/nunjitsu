# Nunjitsu documentation

These guides explain how Nunjitsu works, what it supports, and how to maintain
it. Start with the root [README](../README.md) if you are evaluating or using
the package.

## Guides

| Guide | Read this when you want to… |
| --- | --- |
| [Architecture](architecture.md) | Understand the render pipeline, closed interpreter, and package structure |
| [Security](security.md) | Evaluate the threat model, safe-value rules, capabilities, and resource limits |
| [Compatibility](compatibility.md) | Check supported Nunjucks syntax and intentional differences |
| [Testing](testing.md) | Run or extend the test suites and benchmarks |
| [Releasing](releasing.md) | Prepare, create, and publish a release |

The detailed normative implementation constraints used by coding agents and
maintainers live in [`CONTEXT.md`](../CONTEXT.md). They are intentionally kept
out of these guides so the human documentation stays readable.
